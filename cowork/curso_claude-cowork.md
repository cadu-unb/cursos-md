## VISÃO GERAL DO NOVO PARADIGMA

### De "Chat + Tarefas" para "Parceiro de Projeto"

O Claude Cowork não é apenas uma ferramenta que você **usa**. É um parceiro que você **trabalha junto**.

**Diferença Fundamental:**

| **Chat Comum** | **Projeto Cowork (Colaborativo)** |
|---|---|
| Pergunta → Resposta | Projeto → Contexto Persistente → Artefatos → Iteração |
| Sem memória entre conversas | Claude "lembra" de tudo do projeto |
| Ideal para dúvidas isoladas | Ideal para trabalhos complexos e multietapas |
| "Me explique X" | "Vamos construir X juntos" |

### Descrição

**Objetivos da Aula:**
- Entender que Claude Cowork é um **parceiro de trabalho**, não uma máquina de executar comandos
- Reconhecer quando usar Chat isolado vs. quando criar um Projeto estruturado

**Habilidades Esperadas:**
- Diferenciar tarefas que cabem em Chat vs. tarefas que exigem Projeto
- Reconhecer sinais de quando é hora de escalar para um Projeto
- Adotar mentalidade de "parceria iterativa"

### Conteúdo

#### O Que é um "Coworking" com IA?

Coworking significa **trabalhar junto**. Quando você trabalha com um colega humano:

1. **Vocês têm um objetivo comum** (preparar material pedagógico, organizar currículo)
2. **Há contexto compartilhado** (regras, tons, estilos que ambos conhecem)
3. **O trabalho é iterativo** (versão 1 → feedback → versão 2 → feedback → versão final)
4. **Há responsabilidades** (você não abandona o colega; ele não abandona você)

**Claude Cowork através de "Projects" funciona assim:**

```
Você cria um PROJETO
    ↓
Você alimenta com CONTEXTO (PDFs, documentos, instruções)
    ↓
Claude "lê tudo" e entra no personagem/tom do projeto
    ↓
Você trabalha JUNTO com Claude em ARTEFATOS (documentos vivos)
    ↓
Você dá FEEDBACK em tempo real
    ↓
Claude ajusta especificamente onde você pediu
    ↓
Projeto evolui até ficar pronto
```

**Analogia Prática:**
Imagine que você é professora de Português e vai criar uma apostila com atividades de redação para o 8º ano. 

Com **Chat isolado**, você pediria:
```
"Crie 5 exercícios de redação para 8º ano."
Claude cria. Pronto.
```

Com **Projeto (Cowork)**, você faria:
```
1. Cria um Projeto chamado "Apostila Redação 8º Ano"
2. Adiciona na Knowledge Base:
   - Currículo da escola (que habilidades treinar)
   - 3 exemplos de redações anteriores (tom esperado)
   - Suas instruções: "Exercícios devem ter dificuldade progressiva"
3. Pede ao Claude: "Crie exercício 1"
4. Claude cria → você revisa → "Gosto, mas reduza as instruções"
5. Claude ajusta especificamente
6. Pede exercício 2 (Claude já "conhece" o padrão)
7. 10 minutos depois: apostila inteira, consistente, ajustada
```

**Por que Projeto é melhor para trabalho complexo?**

1. **Coerência** - Claude mantém tom/estilo ao longo de todo o projeto
2. **Eficiência** - Você não repete instruções básicas sempre
3. **Rastreabilidade** - Tudo fica salvo em um só lugar
4. **Evolução** - Versões antigas de artefatos continuam acessíveis

## 1.2 Chat Rápido vs. Projeto Estruturado: Quando Usar Cada Um?

### 📸 Sugestão de Prints
- Tela mostrando decisão: árvore de decisão visual (Tarefa simples? → Chat | Tarefa complexa? → Project)
- Exemplo de Chat com resposta simples vs. Project com múltiplos Artifacts

### Descrição

**Objetivos da Aula:**
- Treinar o julgamento de quando escalar de Chat para Projeto
- Economizar tempo usando a ferramenta certa
- Evitar frustrações por má escolha de abordagem

**Habilidades Esperadas:**
- Reconhecer padrões de "tarefa que merece Projeto"
- Decidir rápido entre Chat e Project sem análise excessiva

### Conteúdo

#### Regra Simples

**Use CHAT quando:**
- A tarefa cabe em **uma pergunta**
- Você quer **resposta rápida**, não iteração
- Não há **contexto complexo** envolvido
- O resultado é **descartável** após uso

Exemplos:
- "Qual é a diferença entre análise sintática e análise semântica?"
- "Sugira uma atividade para quebra-gelo em aula de inglês"
- "Revise esta frase para mim"
- "Explique fração usando analogia com pizza"

**Use PROJECT quando:**
- A tarefa tem **múltiplas etapas** ou **versões**
- Você precisa que Claude **mantenha contexto** entre conversas
- O resultado é **reutilizável** (apostila, currículo, plano anual)
- Você quer **feedback iterativo** (versão 1 → ajusta → versão 2)
- O trabalho envolve **múltiplos documentos/artifacts**

Exemplos:
- Criar apostila completa (20 exercícios, coerentes)
- Montar currículo integrado de disciplina (alinhando BNCC, habilidades, atividades)
- Desenvolver avaliação bimestral com gabarito e rubricas
- Organizar projeto interdisciplinar (Historia + Português + Arte)
- Documentar modelo pedagógico da escola (definições, princípios, exemplos)

## 1.3 Etiqueta de Coworking com IA: Princípios

### 📸 Sugestão de Prints
- Infográfico com 6 princípios (imagens: diálogo, feedback, respeito, revisão, comunicação clara, confiança)
- Screenshot de um Projeto com "Instruções Customizadas" visível

### Descrição

**Objetivos da Aula:**
- Estabelecer normas de comportamento para trabalho eficaz com IA
- Criar cultura de "respeito e responsabilidade mútua" com parceiro IA

**Habilidades Esperadas:**
- Comunicar-se de forma clara e direta com Claude
- Dar feedback construtivo (não punitivo)
- Revisar criticamente o trabalho do parceiro

### Conteúdo

#### 6 Princípios de Etiqueta no Coworking com IA

> **Princípio 1: Clareza acima de tudo**
>
> Seu parceiro IA é tão bom quanto suas instruções. Se você é vago, Claude será vago.
>
> ❌ "Crie uma atividade de matemática"
> ✅ "Crie exercício de divisão com 3 dígitos (nível 6º ano), progressão do fácil para difícil, com números realistas (preços, distâncias)"

> **Princípio 2: Feedback construtivo, não reclamação**
>
> Quando Claude errar, não diga "isso está ruim". Diga o que faltou e como corrigir.
>
> ❌ "Isso não presta."
> ✅ "Bom começo! Mas precisa de mais exemplos práticos. Adicione 2 casos reais de empresas que usam essa estratégia."

> **Princípio 3: Supervisão ativa**
>
> Você é responsável final. Revise SEMPRE o trabalho antes de usar. Não confie cegamente.
>
> ✅ Hábito: "Claude criou → Eu leio → Eu sinalizo erros → Claude corrige → Eu reviso novamente"

> **Princípio 4: Contexto é o rei**
>
> Quanto mais contexto você der (PDFs, exemplos, regras), melhor Claude trabalha.
>
> ❌ "Crie aula de ciências" (Claude não sabe: que série? que matéria? que estilo?)
> ✅ "Crie aula de ciências seguindo este currículo [upload PDF], mantendo tom de [seu documento de estilo], para 7º ano, 45 min"

> **Princípio 5: Iteração rápida, não perfeccionismo paralisante**
>
> Versão 1 nunca é perfeita. Mande Claude criar versão 1, depois refine.
>
> Ruim: Você tenta escrever instruções "perfeitas" por 2 horas
> Bom: Você dá instruções 70% claras, Claude cria, você refina em 10 min

> **Princípio 6: Responsabilidade mútua**
>
> Claude responsável: executar o melhor possível conforme instruções
> Você responsável: revisar, dar feedback, proteger dados sensíveis, uso ético

# MÓDULO 2 — **Conhecimento Compartilhado: Construindo a Base de Conhecimento do Projeto**

## 2.1 O Que É Knowledge Base (Base de Conhecimento)?

### 📸 Sugestão de Prints
- Painel de "Knowledge Base" vazio (com botão "+ Adicionar Arquivo")
- Exemplo de Knowledge Base preenchida (mostrando PDFs, documentos, planilhas adicionados)
- Indicador visual de quanto "contexto" o Claude tem acesso (ex: "300 páginas em contexto")

### Descrição

**Objetivos da Aula:**
- Entender que Knowledge Base é a "memória longa" do projeto
- Aprender a alimentar o Claude com contexto que ele precise
- Reconhecer que qualidade de KB = qualidade de respostas

**Habilidades Esperadas:**
- Identificar quais documentos devem estar na KB
- Organizar arquivos de forma lógica
- Verificar se Claude está "lendo" a KB corretamente

### Conteúdo

#### O Que Entra na Knowledge Base?

Knowledge Base é um espaço onde você coloca **tudo que Claude precisa saber** para trabalhar bem no seu projeto.

**Analogy:**
Imagine que Claude é novo na sua escola. Ele não sabe:
- Qual é o currículo que você segue
- Que tom pedagógico você usa (formal? descontraído?)
- Quais são as regras da instituição
- Que público você atende
- Que estilo de exercício funciona bem

Você coloca tudo isso na Knowledge Base. Agora Claude é **especialista** no seu contexto.

#### Exemplos de O Que Adicionar à KB

**Para Projeto: "Apostila Frações 6º Ano"**

Adicione:
1. **Currículo oficial** (BNCC ou currículo estadual em PDF)
2. **Documento de estilo da escola** (tom esperado, formatação, valores)
3. **Exemplos de suas atividades anteriores** (para Claude copiar o padrão)
4. **Planilha com habilidades** (o que cada aluno deve conseguir fazer)
5. **Documento de diferenças individuais** (alunos com dislexia, deficiência visual, alunos superdotados — como adequar?)

**Para Projeto: "Currículo Integrado de Ciências"**

Adicione:
1. **BNCC de Ciências completo**
2. **Projetos interdisciplinares anteriores** (para inspirar)
3. **Cronograma escolar da escola**
4. **Diretrizes para relatórios de avaliação**
5. **Documentação sobre recursos disponíveis** (laboratório? equipamentos?)

#### Estrutura Recomendada para KB

```
📁 Apostila Frações 6º Ano
├── 📄 01_BNCC_Matematica.pdf
├── 📄 02_Estilo_Pedagogico_Escola.docx
├── 📄 03_Exemplos_Atividades_Anteriores.pdf
├── 📄 04_Habilidades_6ºAno.xlsx
├── 📄 05_Acessibilidade_Turma.docx
└── 📄 06_Tom_e_Linguagem_Esperado.docx
```

Claude terá acesso a tudo isso ao trabalhar com você.

#### Limite de Tamanho e Organização

**Importante:**
- Claude pode processar até 200.000 tokens por interação (= ~150 páginas de texto)
- Se sua KB for > 200 páginas, **organize em categorias**
- Use nomes de arquivo CLAROS e NUMERADOS (01, 02, 03...)
- Prefira PDFs limpos a imagens de texto (OCR funciona, mas é menos preciso)

### Conteúdo

#### Tipos de Arquivo Suportados

| **Formato** | **Ideal para** | **Notas** |
|---|---|---|
| **PDF** | Documentos oficiais, PDFs digitalizados | Melhor suporte; Claude lê textos e tabelas |
| **DOCX** | Documentos editados (Word) | Bom; preserva formatação |
| **TXT** | Textos simples | Rápido, sem ruído |
| **XLSX/CSV** | Dados, planilhas | Claude consegue interpretar |
| **PNG/JPG** | Imagens com texto (cartazes, tabelas, gráficos) | Funciona, mas menos preciso que PDF |
| **MD (Markdown)** | Documentação estruturada | Excelente para notas estruturadas |

#### Passo-a-Passo: Adicionar Seu Primeiro Arquivo

**Suponha que você tem um PDF com o Currículo de Matemática da BNCC:**

1. Dentro do seu Projeto "Apostila Frações 6º Ano", procure a seção **Knowledge Base** (barra esquerda)

2. Clique em **"+ Adicionar Arquivo"** (ou "Add File")

3. Você tem duas opções:
   - **Drag & Drop:** Arraste o arquivo PDF para a janela (mais rápido)
   - **Selecionar:** Clique e navegue pelo seu computador

4. Selecione o PDF do Currículo

5. Espere 5-30 segundos (depende do tamanho)

6. Você verá:
   ```
   ✅ BNCC_Matematica.pdf
   Carregado (45 páginas, 2.3 MB)
   ```

7. **Teste se funciona:**
   - Abra o campo de Chat do Projeto
   - Escreva:
     ```
     Resuma em 3 pontos as habilidades esperadas 
     em frações para o 6º ano segundo a BNCC.
     ```
   - Se Claude responder mencionando conceitos do PDF, ✅ funcionou!

**1. Nomeie arquivos com NÚMEROS e PALAVRAS-CHAVE**

```

anotacoes.docx
tabela.xlsx
```

✅ Bom:
```

04_Habilidades_6ºAno.xlsx

```

USAR para garantir alinhamento com objetivos nacionais.
```

**Benefício:** Você (e Claude) sabe por que o arquivo está ali.

**3. Agrupe por Tipo (se KB ficar grande)**

```

├── 01_BNCC_Matematica.pdf

📁 EXEMPLOS PRÁTICOS
├── 05_Atividades_Boas_Anteriores.pdf
├── 06_Rubrica_Avaliacao.xlsx

📁 CONTEXTO ESPECIAL
├── 07_Acessibilidade.docx
├── 08_Turma_Perfil.docx

Claude lerá na ordem que você colocar. Recomende: do geral para o específico.

# MÓDULO 3 — **Instruções Customizadas: Dando Personalidade ao Seu Parceiro IA**

## 3.1 Custom Instructions: O "Manual de Operação" do Seu Parceiro

### 📸 Sugestão de Prints
- Painel de **Custom Instructions** vazio (com campo "Instruções do Projeto" grande)
- Exemplo de Custom Instructions preenchidas (4-5 pontos bem estruturados)

### Descrição

**Objetivos da Aula:**
- Definir "personalidade" e "regras" que Claude deve seguir no projeto
- Criar consistência ao longo de todas as interações

**Habilidades Esperadas:**
- Escrever instruções claras e não-contraditórias
- Testar se Claude está seguindo instruções
- Iterar e refinar instruções conforme necessário

### Conteúdo

#### O Que São Custom Instructions?

Custom Instructions são como um **"Manual de Operação"** que você deixa visível para o Claude ler sempre que trabalha em seu projeto.

**Exemplo de diferença:**

❌ **Sem Custom Instructions:**
```
Você: "Crie exercício de frações."
Claude: [cria algo genérico]

Você: "Crie outro, mas mais pedagógico."
Claude: [cria novo]

Você: "E este aqui, por favor, siga meu estilo."
Claude: [cria terceiro, mas sem coerência com anteriores]
```

✅ **Com Custom Instructions:**
```
[Você adiciona na seção "Custom Instructions"]

"Crie exercícios de matemática de forma pedagógica:
- Use linguagem acessível (6º ano)
- Progresse de fácil para difícil
- Sempre inclua contexto prático (mercado, tempo, distância)
- Use tom amigável e encorajador"

---

Você: "Crie exercício 1 de frações."
Claude: [cria SEM você ter que repetir instruções]

### Conteúdo

#### Teste 1: Conformidade com Instruções

**Setup:**
Você tem Custom Instructions dizendo:
```
"Todos os exercícios devem ter contexto prático.
NÃO crie exercícios abstratos de matemática."
```

**Teste:**
Peça: "Crie 3 exercícios de divisão."

**Verificação:**
- Leia os 3 exercícios
- Para cada um, pergunte: "Tem contexto prático?"
  - ✅ "48 maçãs em 6 cestas = ?" (contexto: frutas, cestas)
  - ❌ "144 ÷ 12 = ?" (abstrato, sem contexto)

**Se Claude falhar:**
A instrução não foi clara o suficiente. Refine.

---

**Instrução ANTES (vaga):**
```
"Use contexto prático."
```

**Instrução DEPOIS (clara):**
```
"Cada exercício deve ter UMA situação real:
- Compra/venda (preço ÷ quantidade)
- Compartilhamento (chocolates ÷ pessoas)
- Tempo (horas ÷ tarefas)
- Distância (km ÷ dias)

Exemplo BOM: 'João tem 60 figurinhas para repartir entre 5 amigos.'
Exemplo RUIM: '60 ÷ 5 = ?'"
```

---

#### Teste 2: Identificar Contradições

Às vezes, instruções se contradizem:

```
Instrução A: "Crie exercícios fáceis para começar"
Instrução B: "Progresso deve ser rápido"
```

(Contradição: fácil vs. rápido? Fácil demora, rápido é desafiador)

**Como encontrar:**

Você identifica quando Claude **hesita** ou **se desculpa**:

```
Claude: "Eu criei exercícios fáceis conforme você pediu, 
mas tentei fazer progredir rápido também. 
Deu conflito na hora de escolher dificuldade..."
```

**Solução:**
Reescreva de forma coerente:

```
"Crie sequência de 5 exercícios:
1. Exercícios 1-2: Fáceis (garantir confiança)
2. Exercícios 3-4: Moderados (introduzir desafio)
3. Exercício 5: Desafiador (consolidar aprendizado)

Progressão deve ser SUAVE (não saltos de dificuldade)."
```

---

#### Teste 3: Validação com Amostra

**Procedimento:**

1. Peça ao Claude: "Crie 5 exercícios seguindo minhas instruções"

2. Você avalia cada um com checklist:

```
[ ] Exercício 1:
    [ ] Tem contexto prático? SIM/NÃO
    [ ] Linguagem é clara? SIM/NÃO
    [ ] Progride conforme esperado? SIM/NÃO
    [ ] Segue formato definido? SIM/NÃO

[ ] Exercício 2: [mesmo checklist]
...
```

3. Se 80%+ passam: ✅ Instruções estão boas

4. Se < 80%: ❌ Refine instruções e teste novamente

❌ Feedback vago:
```
"Isso não ficou bom."
"Muda tudo."

```

✅ Feedback específico:

De: 'Represente a fração que Maria comeu'

1. Maria comeu 1 pedaço
2. Pizza tinha 4 pedaços
3. Logo, fração = 1/4"
```

## 5.2 Comunicação Estratégica: Como Pedir Tarefas Complexas

### 📸 Sugestão de Prints

**Habilidades Esperadas:**

### Conteúdo

#### Estrutura de Prompt Eficaz para Projetos

Use este template sempre:

```
CONTEXTO (15% do prompt):
"Estou criando apostila de frações para 6º ano.
Você já leu minha KB com exemplos e currículo (lido?).
Segue minhas Custom Instructions sobre tom e formato."

TAREFA ESPECÍFICA (20% do prompt):
"Preciso que você crie os exercícios 6-10 (próximo lote)."

DETALHES/RESTRIÇÕES (40% do prompt):
"Critérios:
- Exercícios 6-8: Fáceis-moderados (tipo pizza/chocolate compartilhado)

- Progressão: não pula dificuldade
- Cada um com enunciado, instrução, espaço resposta, gabarito
- Manter PADRÃO dos exercícios 1-5 (compare com Artifact anterior)"

ENTREGA (15% do prompt):
"Crie um ARTIFACT com todos os 5 exercícios numerados."
```

**Ratios:**
- Contexto: 15%
- Tarefa: 20%
- Detalhes: 40%
- Entrega: 15%
- Flexibilidade: 10%

#### Exemplo Real

❌ **Ruim (vago):**
```
"Crie os exercícios 6 a 10."
```

✅ **Excelente (estruturado):**
```
Estou criando apostila de frações para 6º ano. 
Você já leu minha KB com exemplos e estou seguindo 
as Custom Instructions de tom pedagógico e formato.

Criei exercícios 1-5 (muito fáceis) com sucesso. 
Agora preciso de exercícios 6-10.

PROGRESSO ESPERADO:
- Ex 6-8: Fácil-moderado (alunos já entendem 1/2, 1/4, 1/3)
- Ex 9-10: Moderado-desafio (combinam frações com situações mais complexas)

PADRÃO A MANTER:
- Contexto prático: comida, preço, tempo ou distância
- Números realistas (não crie "7/13", use 1/2, 1/3, 2/3, 1/4, 3/4, etc.)
- Enunciado 1-2 frases, instrução 1 frase, espaço para resposta, gabarito breve

ENTREGA:
Crie um ARTIFACT com exercícios 6-10 seguindo o padrão dos anteriores.
Cite qual exercício é qual nível de dificuldade.
```

**Diferença:** No segundo, Claude sabe exatamente o que fazer. Menos iterações.
