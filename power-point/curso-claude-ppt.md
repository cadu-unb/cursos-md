# Visão Geral do Curso Reformulado

O Claude para PowerPoint é mais do que um gerador de slides. É um **parceiro criativo** que você pode interrogar, direcionar e refinar para produzir apresentações de qualidade profissional em fração do tempo tradicional.

### Analogia Central

> Imagine que você contratou um roteirista e designer experiente. Você descreve a ideia, o público e os objetivos. O Claude estrutura tudo: sequência lógica, hierarquia visual, notas para sua fala, até sugere cores e estilos. Você revisa, ajusta e aprova. O resultado? Uma apresentação que parece ter levado semanas, mas foi criada em horas.

| "PowerPoint trava ao usar Claude" | Feche abas/extensões do navegador que possam consumir memória. |
| "Internet lenta; Claude demora" | Simplifique o prompt. Textos muito longos podem levar mais tempo. |

# Módulo 1 — **O Claude Entrou na Sala: Entendendo Seu Novo Parceiro Criativo**

### Objetivo da Aula
Estabelecer uma mentalidade correta sobre o que Claude é (e o que não é) no contexto de criação de apresentações.

### Habilidades Esperadas
- Compreender o papel do Claude como assistente criativo, não substituto
- Identificar tarefas onde Claude agrega valor máximo
- Reconhecer limitações e quando intervir manualmente
- Desenvolver a "mentalidade de colaboração" com IA

---

### 1. O Novo Paradigma: Claude Como Arquiteto e Roteirista

#### A Ilusão Antiga
"Claude gera slides. Eu copiar e colar. Pronto!"

❌ **Resultado:** Slides genéricos, sem personalidade, que parecem "feitos por IA"

#### A Realidade Nova
"Claude estrutura ideias. Eu refinamento. Nós criamos algo único."

✅ **Resultado:** Apresentações memoráveis, com voz do professor, visualmente coerentes

---

### 2. Mapa de Capacidades: Onde Claude Excela

📸 **Sugestão de Print:** Infográfico com 4 quadrantes:
- **Quadrante Verde (Máximo Valor):** Estrutura narrativa, roteiros de fala, hierarquia de conteúdo
- **Quadrante Amarelo (Valor Médio):** Geração de textos iniciais, sugestões de design
- **Quadrante Vermelho (Baixo Valor):** Design gráfico (use Canva/Designer), edição final de imagens
- **Quadrante Cinza (Não Faz):** Criar imagens reais (use DALL-E, Midjourney), garantir dados 100% precisos

---

### 3. Os 5 Superpoderes do Claude para Apresentações

#### 1️⃣ Roteirista: Estrutura Lógica

Claude pode transformar:
```md
ENTRADA: "Preciso falar sobre ciclo da água para crianças de 6º ano"
SAÍDA: Sequência de 6 slides com abertura, desenvolvimento gradual, exemplo prático, atividade, fechamento
```

#### 2️⃣ Curador: Simplifica Conteúdo Denso

Claude pode transformar:
```md
ENTRADA: Texto acadêmico de 2000 palavras sobre mitose celular
SAÍDA: 5 bullet points claros + 2 analogias simples + 1 pergunta provocativa
```

#### 3️⃣ Roteirista de Fala: Expande Títulos em Narrativas

Claude pode transformar:
```md
ENTRADA: Título do slide "A importância da biodiversidade"
SAÍDA: Roteiro de fala natural de 2 minutos com tons de conversa, exemplos locais, pausas para perguntas
```

#### 4️⃣ Estrategista Visual: Sugere Estruturas e Layouts

Claude pode transformar:
```md
ENTRADA: "Preciso comparar 3 métodos de aprendizado"
SAÍDA: Sugestão de layout (3 colunas lado a lado, ou timeline, ou Venn diagram)
```

#### 5️⃣ Gerador de Prompts: Descreve Imagens para Bancos de Dados

Claude pode transformar:
```md
ENTRADA: "Preciso uma imagem de um ecossistema saudável"
SAÍDA: Descrição detalhada para usar no Unsplash, Pexels ou DALL-E
```

---

### 4. O Que Claude NÃO Faz (E Está Tudo Bem)

❌ **NÃO cria imagens** (use DALL-E, Midjourney, Unsplash)  
❌ **NÃO garante precisão histórica** (você valida)  
❌ **NÃO conhece seus alunos** (você personaliza)  
❌ **NÃO tem gosto artístico objetivo** (você decide a estética)  
❌ **NÃO entende contexto cultural específico** (você adapta)

---

### 5. A Mentalidade de Colaboração: Você + Claude

📸 **Sugestão de Print:** Diagrama de ciclo:
```
Você (Ideia + Objetivo) → Claude (Estrutura + Rascunho) 
→ Você (Refinamento + Personalização) → Claude (Iteração)
→ Você (Aprovação + Ajustes Finais) → Resultado Profissional
```

---

### 6. Exemplo Prático: O Poder da Colaboração

**Cenário Real:**

Professora de Biologia precisa criar apresentação sobre **fotossíntese** para 8º ano.

**Passo 1 — Você define o objetivo:**
```md
Vou criar uma aula sobre fotossíntese. Alunos têm 13 anos e adoram biologia. 
Objetivo: entender as 3 etapas principais. Preciso de 6 slides, com muita interatividade.
```

**Passo 2 — Claude estrutura:**
```md
Sugestão de sequência:
1. Abertura: "O que as plantas comem?" (pergunta provocativa)
2. Resposta: "Elas fazem seu próprio alimento"
3. Etapa 1: Absorção de luz
4. Etapa 2: Transformação em energia
5. Etapa 3: Produção de oxigênio
6. Fechamento: Atividade interativa (quiz)
```

**Passo 3 — Você personaliza:**
"Adicione um exemplo de uma planta no jardim da escola. Coloque uma pergunta final que desafie eles a pensarem sobre poluição do ar."

**Passo 4 — Claude refina:**
```md
Sugestão nova para Slide 5:
"E se a poluição do ar bloqueasse a luz? O que aconteceria com a fotossíntese?"
[Deixar espaço para resposta dos alunos]
```

**Passo 5 — Você aprova e implementa:**
Resultado: Apresentação única, com sua voz, com referências locais, pronta para aula.

**⏱️ Tempo total: 15 minutos** (em vez de 1 hora fazendo do zero)

# Módulo 2 — **Antes de Pedir: Preparando Seu Projeto**

### Objetivo da Aula
Estruturar informações e definir objetivos claros ANTES de enviar prompts ao Claude.

### Habilidades Esperadas
- Definir objetivo pedagógico com clareza
- Identificar público-alvo (idade, conhecimento prévio, contexto)
- Estruturar conteúdo em hierarquia de informações
- Criar um "brief" ou resumo executivo para compartilhar com Claude

---

### 1. O Briefing: Sua Bússola

Antes de qualquer prompt, responda estas 7 perguntas:

```md
1. OBJETIVO: O que quero que alunos aprendam ou façam após a aula?
2. PÚBLICO: Qual é a idade, nível escolar, conhecimento prévio?
3. CONTEXTO: Onde isso se encaixa no currículo? É primeira aula ou revisão?
4. DURAÇÃO: Quantos minutos terei para apresentar?
5. FORMATO: Quantos slides? Há limitações técnicas (internet lenta, sem áudio)?
6. ATIVIDADE: Que atividade ou pergunta vai engajar alunos?
7. CONSTRAINTS: Há tópicos proibidos, palavrões a evitar, ou formatos específicos?
```

📸 **Sugestão de Print:** Template preenchido com exemplo (ciclo da água, 6º ano, 20 minutos, etc.)

---

### 2. Estruturação de Conteúdo: Pirâmide de Informação

O Claude respeita hierarquia. Se você estruturar assim:

```md
TÓPICO GERAL: Alimentação Saudável

├─ CONCEITO PRINCIPAL 1: Grupos de Alimentos
│  ├─ Detalhes: Proteínas, Carboidratos, Gorduras
│  └─ Exemplos: Frango (proteína), Pão (carboidrato)
│
├─ CONCEITO PRINCIPAL 2: Proporções Corretas
│  ├─ Detalhes: Prato saudável é 50% vegetais, 25% proteína, 25% carboidrato
│  └─ Exemplos: [imagem de prato equilibrado]
│
└─ CONCEITO PRINCIPAL 3: Hábitos Diários
   ├─ Detalhes: Hidratação, intervalos entre refeições
   └─ Exemplos: Beber 2 litros de água, comer a cada 3 horas
```

Claude vai respeitar essa estrutura na apresentação.

---

### 3. Persona do Aluno: Conheça Seu Público

Descreva seus alunos com detalhes:

❌ Genérico:
"Para crianças de 6º ano"

✅ Específico:
"Alunos de 6º ano (11-12 anos), escola em periferia de São Paulo, maioria nunca viu um microscópio, 60% tem acesso a smartphone em casa, nível de atenção: 20-25 minutos."

Claude **ajusta linguagem, ritmo e exemplos** baseado nessas informações.

---

### 4. Mapa Mental: Organize Antes de Digitar

# Módulo 3 — **O Prompt é o Bilhete: Pedindo ao Claude de Forma Estratégica**

### Objetivo da Aula
Dominar a arte de escrever prompts que geram resultados de primeira qualidade.

### Habilidades Esperadas
- Estruturar prompts em componentes (contexto, tarefa, formato, restrições)
- Iterar e refinar prompts baseado em resultados
- Usar técnicas de "prompt engineering" simples e eficazes
- Saber quando pedir ajustes vs. recomeçar

---

### 1. Anatomia de um Prompt Excelente

```md
[CONTEXTO] + [TAREFA] + [FORMATO] + [RESTRIÇÕES] = PROMPT PODEROSO
```

#### Componente 1: CONTEXTO
Quem é você? Qual é a situação?

```md
❌ Fraco: "Preciso de slides sobre água"
✅ Forte: "Sou professora de Ciências em escola pública, ensinando para crianças de 6º ano 
que nunca tiveram aula prática de laboratório. Preciso introduzir conceito de ciclo da água."
```

#### Componente 2: TAREFA
O que exatamente você quer que Claude faça?

```md
❌ Fraco: "Faça uma apresentação"
✅ Forte: "Crie uma estrutura de 6 slides que mostre as 4 etapas do ciclo da água 
(evaporação, condensação, precipitação, escoamento)"
```

#### Componente 3: FORMATO
Como você quer o resultado?

```md
❌ Fraco: "Estrutura de apresentação"
✅ Forte: "Para cada slide, forneça:
   1. Título do slide
   2. Bullet points (máximo 4, máximo 10 palavras cada)
   3. Uma atividade ou pergunta para engajar alunos
   4. Sugestão de imagem ou diagrama"
```

#### Componente 4: RESTRIÇÕES
O que evitar? O que é obrigatório?

```md
❌ Fraco: "Mantenha simples"
✅ Forte: "Use linguagem adequada para 11-12 anos (evite jargão científico). 
Inclua pelo menos um exemplo do dia a dia (ex: chuva, roupa molhada secando). 
Evite gráficos muito complexos."
```

---

### 2. Template Universal para Prompts de Apresentação

Use este template como base:

```md
# CONTEXTO
Sou [profissão] em [ambiente]. Meu público é [descrição detalhada do aluno].
Contexto: [onde isso se encaixa no currículo?].

# TAREFA
Preciso que você [ação específica]: [detalhes].

# FORMATO
Estruture o resultado assim:
[descreva exatamente como quer o output]

# RESTRIÇÕES & PREFERÊNCIAS
- Deve: [obrigações]
- Não deve: [proibições]
- Tom: [formal/casual/lúdico/etc]
- Nível de profundidade: [introdutório/intermediário/avançado]

# EXEMPLOS (opcional mas poderoso)
Se possível, indique exemplos do que você quer:
[Mostrar 1-2 exemplos de saída ideal]
```

---

### 3. Técnica: Prompt Interativo e Iterativo

Claude não lê mentes. Você vai precisar de múltiplos turnos:

**Turno 1 — Estrutura Geral:**
```md
Estruture uma apresentação sobre fotossíntese para 8º ano com 5 slides. 
Apenas os títulos e tópicos principais, sem textos completos.
```

Claude responde com versão draft.

**Turno 2 — Refinamento:**
```md
Ótimo! Agora:
1. Adicione uma pergunta provocativa no Slide 1
2. Mude o Slide 3 para incluir um experimento que alunos podem fazer
3. Torne a linguagem mais simples (como se explicasse para um colega, não para um livro)
```

Claude refina.

**Turno 3 — Expansão:**
```md
Agora expanda cada slide com notas de fala de 1-2 minutos. 
Use tom conversacional. Inclua pausas para perguntas.
```

Claude expande.

📸 **Sugestão de Print:** Diálogo tipo screenshot mostrando 3 turnos de prompt/resposta no painel Claude do PowerPoint

---

### 4. Técnicas Avançadas de Prompt

#### 4.1 Chain-of-Thought (Pedir para Claude Pensar em Voz Alta)

```md
Antes de criar os slides, pense em voz alta:
1. Qual é o conceito-chave que alunos precisam entender?
2. Qual é o errado comum que eles têm?
3. Qual é a melhor sequência para desconstruir esse conceito?

Depois, baseado nessa análise, crie a estrutura de 4 slides.
```

#### 4.2 Few-Shot (Mostrar Exemplos)

```md
Aqui estão 2 exemplos de como gosto que bullets sejam escritos:

EXEMPLO 1 — Bom:
"Água evaporada em oceanos sobe (invisível)"

EXEMPLO 2 — Bom:
"Gotículas formam nuvens em altitudes frias"

Agora crie 4 bullets assim para o tema "Condensação".
```

#### 4.3 Role-Playing (Pedir para Claude Assumir um Papel)

```md
Você é um professor de ciências extremamente criativo que consegue explicar tópicos 
complexos usando apenas analogias com coisas do dia a dia dos alunos (comida, celular, esportes, etc).

Crie uma abertura de slide sobre "Mitose" usando 2-3 analogias simples.
```

---

### 5. Sinais de Alerta: Quando o Prompt Não Funciona

| Sinal | Solução |
|-------|---------|
| "Claude respondeu de forma muito genérica" | Seja mais específico sobre público. Adicione exemplos. |
| "A resposta é muito técnica" | Especifique nível de linguagem desejado. Peça para explicar como para crianças. |
| "Faltaram detalhes" | Reformule a tarefa. Quebra em sub-tarefas. |
| "A estrutura não faz sentido" | Mostre um exemplo de estrutura ideal. Use chain-of-thought. |

---

### 6. Exemplo Completo: Prompt Forte vs. Fraco

#### ❌ PROMPT FRACO:
```md
Crie uma apresentação sobre sustentabilidade para 7º ano.
```

**Resultado:** Genérico, sem personalidade, pode não ter atividade interativa

#### ✅ PROMPT FORTE:
```md
# CONTEXTO
Sou professora de Ciências em escola de zona rural do Ceará. Meus alunos (13 anos) 
vêm de famílias de agricultores e pecuaristas. Eles conhecem na prática o problema 
de seca e desertificação. Nunca estudaram sustentabilidade formalmente.

# TAREFA
Crie uma apresentação de 6 slides que conecte sustentabilidade com a realidade deles. 
O objetivo é que vejam sustentabilidade não como "problema global abstrato" mas como 
"algo que afeta a renda da minha família".

# FORMATO
Para cada slide:
- Título (pergunta provocativa de preferência)
- 3-4 bullet points (máximo 8 palavras cada)
- 1 frase conectando ao contexto deles (seca, agricultura, água)
```

# Módulo 4 — **Estruturando a Narrativa: Do Zero ao Deck Coerente**

### Objetivo da Aula
Criar apresentações com uma arco narrativo claro (abertura, clímax, resolução) que prende atenção.

### Habilidades Esperadas
- Aplicar estruturas de narrativa clássicas (pirâmide, jornada do herói, problema-solução)
- Criar "throughline" (linha condutora) que une todos os slides
- Escrever uma abertura memorável e um fechamento impactante
- Usar técnica de "storytelling visual"

---

### 1. As Estruturas de Narrativa Mais Eficazes para Educação

#### 1.1 Estrutura Clássica: Problema → Solução → Ação

```
ABERTURA: Contexto / Pergunta Provocativa
         ↓
DESENVOLVIMENTO: Entender o Problema (detalhes, causas)
         ↓
CLÍMAX: A Solução / A Ideia Central
         ↓
APLICAÇÃO: Como usar isso? Exemplo prático
         ↓
FECHAMENTO: Call-to-action (reflita, experimente, converse)
```

**Exemplo:**
- Slide 1: "Por que as plantas precisam de luz?" (pergunta provocativa)
- Slide 2-3: "O que é fotossíntese e como funciona" (conteúdo denso simplificado)
- Slide 4: "A fórmula: luz + água + CO2 = glicose" (aha moment!)
- Slide 5: "Veja uma folha ao microscópio: estrutura real" (aplicação)
- Slide 6: "Experimento: coloque uma planta no escuro. O que acontece?" (ação)

#### 1.2 Estrutura de Jornada: O Aluno é o Herói

Não: Claude é o herói que vem salvar.  
Sim: Aluno é o herói em jornada de aprendizado.

```
1. O Mundo Comum: "Você já viu uma nuvem?"
2. O Chamado: "Quer entender como ela se forma?"
3. Os Desafios: "Pareça fácil, mas tem 3 etapas"
4. O Aprendizado: "Aqui estão as etapas"
5. O Retorno: "Agora você pode prever quando vai chover"
```

#### 1.3 Estrutura de Curiosidade: Pergunta → Suspense → Resposta

```
Slide 1: "Por que o céu é azul?" (Desperta curiosidade)
Slide 2: "A maioria acha que é..." (Expectativa)
Slide 3: "Mas na verdade..." (Revelação)
Slide 4: "Veja a prova:" (Validação)
Slide 5: "Agora quando olhar para o céu..." (Aplicação pessoal)
```

### 2. A "Throughline": Mantendo o Fio Condutor

Uma boa apresentação tem UMA ideia central. Todos os slides servem para reforçá-la.

**TEMA CENTRAL:** "Como as plantas transformam luz em comida"

### 3. Abertura Memorável: Seus Primeiros 30 Segundos

A abertura decide se alunos prestam atenção ou dormem.

#### Tipos de Abertura Eficazes:

**Tipo 1: Pergunta Provocativa**

```md

```

**Tipo 4: Conexão Pessoal**
```md
"Quem aqui tem planta em casa? Já parou para pensar o que ela está fazendo agora enquanto 
estamos em aula? Ela está fabricando seu próprio alimento do nada."
```

```md
❌ Slide carregado = aluno não sabe aonde olhar

```

### 1. Por Que Notas de Orador São Críticas

**Cenário Ruim:**
- Slide tem 3 bullet points
- Você olha para bullet point
- Mente vai em branco (síndrome do "ah, era pra falar sobre quê mesmo?")
- Pausa desconfortável
- Alunos notam sua insegurança

**Cenário Bom:**
- Slide tem 3 bullet points
- Você olha suas notas de orador com uma narrativa fluida
- Fala naturalmente (parece preparado mas não lê de papel)
- Alunos não percebem que você está seguindo roteiro
- Apresentação flui como conversa

### 2. Exemplo de VBA Simples: Criar um Slide com Texto

```md
Escreva uma macro VBA que:

2. Adiciona um slide com layout "Título e Conteúdo"

4. Insere 3 bullets:
   - Luz é absorvida
   - Água é processada
   - Oxigênio é produzido
```

Claude responde com código como:

```vba
Sub CriarAulaFotossintese()
    ' Cria uma nova apresentação PowerPoint
    Dim pptApp As Object
    Dim pptPres As Object
    Dim pptSlide As Object
    Dim pptShape As Object
    
    ' Inicia PowerPoint
    Set pptApp = CreateObject("PowerPoint.Application")
    pptApp.Visible = True
    
    ' Cria nova apresentação
    Set pptPres = pptApp.Presentations.Add()
    
    ' Adiciona slide com layout "Título e Conteúdo"
    Set pptSlide = pptPres.Slides.Add(1, 3) ' Layout 3 = Title and Content
    
    ' Define o título
    pptSlide.Shapes.Title.TextFrame.TextRange.Text = "Fotossíntese"
    
    ' Define bullets (conteúdo)
    Dim textBox As Object
    Set textBox = pptSlide.Shapes(2).TextFrame ' Forma 2 é o conteúdo
    
    textBox.TextRange.Text = "Luz é absorvida"
    textBox.TextRange.ParagraphFormat.Level = 0
    
    textBox.TextRange.InsertAfter vbNewLine & "Água é processada"
    textBox.TextRange.InsertAfter vbNewLine & "Oxigênio é produzido"
    
    ' Salva a apresentação
    pptPres.SaveAs "C:\MinhaAula_Fotossintese.pptx"
    
    MsgBox "Aula criada com sucesso!"
End Sub
```

**Prompt:**
```md

- Nome da cor
- Código RGB ou Hex
- Onde usar (fundo, títulos, destaque)
- Por que transmite essa emoção
```


# Módulo 8 — **Segurança, Privacidade, Ética e Revisão Humana**

### Objetivo da Aula
Usar Claude responsavelmente, protegendo dados de alunos e mantendo padrões éticos.

### Habilidades Esperadas
- Entender onde Claude processa dados (nuvem, não localmente)
- Saber o que NUNCA compartilhar com Claude
- Validar precisão de informações
- Manter autoria responsável e transparência com alunos

---

### 1. Onde Claude Processa Seus Dados

⚠️ **CRÍTICO:** Claude processa em **servidores remotos** (Anthropic, hospedados em nuvem).

❌ **Seus dados NÃO ficam:**
- Apenas no seu computador
- Criptografados localmente
- "Privados" entre você e PowerPoint

✅ **Seus dados SIM são:**
- Transmitidos para servidor da Anthropic
- Processados lá
- Potencialmente armazenados para melhorias de modelo (dependendo da política)

**Implicação:** Nunca envie dados sensíveis.

---

### 2. O Que NUNCA Compartilhar com Claude

```md
❌ NUNCA:
- Nomes de alunos específicos
- Datas de nascimento / RG / CPF
- Endereços residenciais
- Diagnósticos médicos (ex: "João tem TDAH")
- Notas ou histórico acadêmico identificado
- Fotos ou vídeos de menores
- Informações de NEE (Necessidades Educacionais Especiais)
- Senhas ou dados bancários

✅ PODE:
- Tema genérico: "Como ensinar frações para crianças com dificuldade de aprendizado"
- Estrutura anônima: "Preciso de atividade para alunos com [nível de profundidade]"
- Contexto geral: "Escola em zona rural com internet lenta"
- Feedback genérico: "Alunos têm dificuldade em entender metaphoras"
```

---

### 3. Validação de Informações: O Risco da Alucinação

Claude é uma IA **generativa**. Às vezes "alucina" — inventa dados que parecem reais.

**Exemplo:**
```md
Você: "Quantas células tem uma folha de uma árvore?"

Claude (alucinando): "Uma folha típica de árvore tem cerca de 50 milhões de células epidérmicas."

Realidade: Esse número é inventado. Claude soou confiante mas estava errado.
```

**Defesa:**
1. Sempre verifique dados críticos em **fontes confiáveis** (livro didático, site oficial, cientista)
2. Especialmente para **apresentações para alunos**, valide tudo
3. Se usar informação de Claude na aula, e aluno questionar, esteja preparado para validar

**Prompt para validação:**
```md
Este fato que você me passou está correto? 
"[informação]"

Cite a fonte ou estude onde encontrou este dado. Se não tem certeza, diga "não tenho certeza".
```

---

### 4. Transparência com Alunos: Devo Contar que Usei IA?

**Resposta curta:** Depende, mas tendência é **SIM, contar.**

**Cenário 1: Apresentação criada com ajuda de Claude**
```md
"Alunos, esta apresentação foi criada com ajuda de uma IA chamada Claude. 
Eu criei a estrutura e revisei tudo, mas Claude ajudou com organização e linguagem. 
O conteúdo é cuidadosamente verificado por mim."
```

**Cenário 2: Atividade usada para criar**
```md
"Vocês vão usar uma IA para [tarefa]. É ok usar IA para [X], mas precisam revisar 
e melhorar a saída. Objetivo é aprender a trabalhar com tecnologia."
```

**Cenário 3: Código ou algoritmo gerado**
```md
"Este código foi gerado por IA, depois testei e adaptei para funcionar com vocês."
```

✅ **Benefício:** Alunos aprendem que IA é ferramenta, e você modela uso responsável.

---

### 5. Checklist de Segurança Antes de Usar Apresentação

```md
[ ] Conteúdo foi verificado para precisão (datas, nomes, fatos)?
[ ] Linguagem está adequada para a idade dos alunos?
[ ] Nenhum dado sensível de aluno está na apresentação?
[ ] Imagens têm fontes atribuídas (se necessário)?
[ ] Estrutura faz sentido no contexto da minha aula?
[ ] Testei a apresentação no projetor antes de apresentar?
[ ] Se fiz com ajuda de IA, alunos sabem disso?
[ ] Posso defender cada informação se aluno questionar?
[ ] Slides seguem políticas da escola (sem linguagem questionável, etc)?
```

---

### 6. Ética em Educação: Princípios Fundamentais

| Princípio | Aplicação |
|-----------|-----------|
| **Autenticidade** | Use IA para melhorar sua aula, não para enganar alunos ou colegas |
| **Responsabilidade** | Você é responsável por todo conteúdo que usa, independente se Claude ajudou |
| **Transparência** | Se usou IA, considere contar (especialmente para alunos maiores) |
| **Proteção de Dados** | Nunca exponha dados pessoais de alunos a serviços em nuvem sem consentimento |
| **Acessibilidade** | Considere alunos com deficiência visual, auditiva, etc ao usar IA |
| **Rigor** | Valide informações antes de ensinar; IA erra |
