Você é um designer editorial especializado na estilização de e-books (formato EPUB) utilizando CSS limpo, leve e altamente compatível com e-readers (Kindle, Kobo, Apple Books e e-readers móveis). 

Preciso que você crie um arquivo "estilo.css" completo e generalista para ser injetado via terminal pelo Pandoc na conversão de um arquivo Markdown para EPUB. O livro é um guia técnico/pedagógico.

O CSS deve seguir rigorosamente as seguintes diretrizes de design e compatibilidade:

1. COMPATIBILIDADE DE E-READERS (O mais importante):
- Evite propriedades modernas de layout web como Flexbox, Grid, posições absolutas ou flutuações (floats) complexas. Use o fluxo de bloco padrão.
- Use unidades relativas (em ou %) para tamanhos de fonte, margens e espaçamentos, permitindo que o usuário aumente ou diminua a fonte no aparelho sem quebrar o layout.

2. TIPOGRAFIA E CORPO DO TEXTO (body):
- Defina uma fonte padrão serifada clássica (ex: "Georgia", serif) para o corpo do texto para uma leitura confortável.
- Defina o alinhamento como justificado (`text-align: justify;`) e um espaçamento de linha agradável (`line-height: 1.5;`).
- Adicione margens globais sutis nas laterais (ex: 5%).

3. CAPÍTULOS E SUBTÍTULOS (h1, h2, h3):
- Para os títulos principais (h1), utilize uma fonte sans-serif limpa (ex: "Helvetica", sans-serif) para dar contraste. Centralize-os e adicione a propriedade `page-break-before: always;` para forçar cada capítulo a começar em uma nova página.
- Estilize os h2 e h3 de forma hierárquica e elegante, garantindo que eles fiquem visualmente colados ao texto que os sucede (margem inferior pequena).

4. PARÁGRAFOS E RECUOS DE LIVRO:
- Implemente a regra clássica de design editorial: todo parágrafo comum deve ter um recuo na primeira linha (`text-indent: 1.5em;`), com margens superiores e inferiores zeradas.
- EXCEÇÃO: O primeiro parágrafo logo após um título (h1, h2 ou h3) NÃO deve ter recuo (`text-indent: 0;`). Crie essa regra combinada.

5. ELEMENTOS ESPECÍFICOS DO NOSSO LIVRO (Blocos de Código, Glossário e Breadcrumbs):
- Blocos de código (pre, code): Devem usar uma fonte monospace limpa, com um fundo cinza bem claro, bordas levemente arredondadas e um espaçamento interno (padding) adequado. Garanta que linhas longas de código quebrem ou permitam rolagem para não estourarem a tela do e-reader.
- Breadcrumbs (p > em): Estilize a linha de navegação técnica que fica no topo das subseções (ex: *Ferramenta > Menu > Opção*). Ela deve ter um tamanho menor (ex: 0.8em), cor cinza escura discreta, sem o recuo tradicional de parágrafo, funcionando como uma etiqueta sutil.
- Links do Glossário ([id^="glos-"]): Destaque os termos que possuem link para o glossário com uma cor azul discreta e um sublinhado pontilhado suave, indicando interatividade sem parecer um link de internet poluído.

Por favor, forneça o código CSS puro, amplamente documentado com comentários explicando cada seção.
